# RenoPack Sales Employee Spec v1

هذا الملف يحدد النسخة الأولى العملية من دور `موظف المبيعات` داخل RenoPack، بحيث لا يكون مجرد مستخدم يرى بعض صفحات الأدمن، بل مسار تشغيل مستقل له بياناته، مهامه، وتقاريره الخاصة.

## 1. الهدف

موظف المبيعات مسؤول عن:

- إدارة قاعدة العملاء المكلف بها
- إدارة قاعدة الورش المكلف بها
- المتابعة اليومية عبر الاتصال أو الزيارة أو واتساب
- تحويل العميل أو الورشة إلى فرصة فعلية
- تحويل الفرصة إلى طلب أو ورشة منضمة
- رفع تقرير يومي ومتابعة هدفه الشخصي

هذا الدور **لا** يرى البيانات المالية الكاملة للإدارة، ولا يرى كل عملاء النظام، ولا كل الورش، إلا ما تم تعيينه له.

## 2. مبادئ الصلاحيات

موظف المبيعات يرى فقط:

- العملاء المعيّنين له
- الورش المعيّنة له
- المهام الخاصة به
- تقاريره الشخصية
- الطلبات الناتجة من عملائه أو ورشه

ولا يرى:

- المصروفات
- أرباح الورش
- كل المستخدمين
- كل العملاء
- كل الطلبات
- إعدادات الإدارة

## 3. الشاشات المطلوبة

### 3.1 Sales Dashboard

تعرض:

- عدد العملاء المعيّنين له
- عدد العملاء الجدد اليوم
- عدد المتابعات المطلوبة اليوم
- عدد العملاء الذين تم تحويلهم لطلبات
- عدد الورش التي تم ضمها أو متابعتها
- نسبة التحويل
- هدف اليوم / الأسبوع
- آخر المهام المفتوحة
- آخر التحويلات الناجحة

### 3.2 My Customers

قائمة العملاء الخاصة بالموظف.

الأعمدة الأساسية:

- الاسم
- الهاتف
- المنطقة
- السيارة
- حالة العميل
- مصدر العميل
- آخر تواصل
- موعد المتابعة القادمة
- الموظف المسؤول
- عدد مرات التواصل

الإجراءات:

- إضافة عميل جديد
- تعديل بيانات العميل
- تسجيل محاولة تواصل
- جدولة متابعة
- تحويل العميل إلى طلب
- إعادة تعيين العميل لمدير/موظف آخر بواسطة الإدارة فقط

### 3.3 My Workshops

قائمة الورش الخاصة بالموظف.

الأعمدة الأساسية:

- اسم الورشة
- الهاتف
- المنطقة
- المسؤول
- حالة الورشة
- آخر متابعة
- موعد المتابعة القادمة
- الموظف المسؤول
- نتيجة آخر زيارة أو مكالمة

الإجراءات:

- إضافة ورشة جديدة
- تعديل بيانات الورشة
- تسجيل متابعة
- رفعها كطلب انضمام
- تحويلها إلى ورشة منضمة

### 3.4 Tasks

قائمة المهام اليومية الخاصة بالموظف.

أنواع المهام:

- اتصال
- زيارة
- متابعة واتساب
- متابعة عميل
- متابعة ورشة
- استكمال بيانات
- إغلاق فرصة

حالات المهمة:

- `pending`
- `in_progress`
- `completed`
- `postponed`
- `cancelled`

الحقول:

- عنوان المهمة
- النوع
- العميل أو الورشة المرتبطان بها
- المنطقة
- وقت التنفيذ
- الملاحظات
- الحالة
- النتيجة

### 3.5 Daily Report

نموذج تقرير يومي يكتبه الموظف.

يتضمن:

- عدد العملاء الجدد
- عدد المكالمات
- عدد الزيارات
- عدد الورش التي تم التواصل معها
- عدد العملاء الذين أبدوا اهتمامًا
- عدد التحويلات إلى طلب
- المعوقات
- خطة الغد

### 3.6 My Performance

تقرير أداء شخصي وليس تقرير إدارة عام.

يتضمن:

- عدد العملاء الذين أضافهم
- عدد العملاء النشطين لديه
- عدد العملاء الذين تم تحويلهم لطلبات
- قيمة الطلبات الناتجة عن عمله
- عدد الورش التي أدخلها
- عدد الورش التي انضمت فعليًا
- نسبة التحويل
- معدل إنجاز المهام
- الأداء اليومي / الأسبوعي / الشهري

## 4. دورة العمل

### 4.1 دورة العميل

1. يتم إدخال العميل بواسطة:
   - موظف المبيعات
   - أو موظف إدخال البيانات
2. يتم تعيين العميل لموظف مبيعات محدد
3. يبدأ الموظف المتابعة
4. يتم تحديث حالة العميل بعد كل تواصل
5. عند الجاهزية يتم تحويله إلى طلب فعلي
6. يُسجل التحويل على الموظف صاحب العملية

### 4.2 دورة الورشة

1. يتم إدخال الورشة
2. تُعيّن لموظف مبيعات
3. يتم التواصل وزيارات الإقناع
4. تسجل نتائج المتابعة
5. يتم رفع طلب انضمام
6. تتحول إلى ورشة معتمدة عند الموافقة
7. يسجل الإنجاز على الموظف

## 8. ما يحتاجه المدير أو الأدمن

الإدارة تحتاج:

- تعيين العملاء لموظف
- تعيين الورش لموظف
- نقل عميل من موظف لآخر
- نقل ورشة من موظف لآخر
- إنشاء مهام للموظف
- رؤية تقارير الأداء المقارنة بين الموظفين

## 9. ما لا يجب استخدامه لموظف المبيعات

لا يجب أن نعتمد على صفحات الإدارة الحالية كحل نهائي:

- `/admin/sales`
- `/admin/orders`
- `/admin/users`
- `/admin/expenses`

يمكن الاستفادة من بعض البيانات منها مؤقتًا، لكن يجب إنشاء صفحات `Sales` مستقلة.

## 10. ترتيب التنفيذ المقترح

### المرحلة 1

- تثبيت صلاحيات موظف المبيعات الحالية
- إخفاء كل صفحات الإدارة غير المناسبة
- إنشاء `Sales Dashboard`

### المرحلة 2

- إنشاء `My Customers`
- إنشاء `My Workshops`
- إنشاء assignment logic

### المرحلة 3

- إنشاء `Tasks`
- إنشاء `Daily Report`
- إنشاء `My Performance`

### المرحلة 4

- ربط التحويلات الفعلية بالطلبات والورش
- تقارير مقارنة بين موظفي المبيعات

## 11. القرار التنفيذي

دور `sales employee` يجب أن يتحول من:

- موظف يرى بعض صفحات الأدمن

إلى:

- نظام CRM + Follow-up + Task Management + Conversion Tracking

وهذا هو الأساس الصحيح لقياس:

- أفضل موظف مبيعات
- أفضل منطقة
- أفضل مصدر عميل
- أفضل مسار تحويل
